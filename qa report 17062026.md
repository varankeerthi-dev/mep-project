# QA Report (Static Codebase Analysis)
**Date:** June 17, 2026
**Project:** mep-project

## Executive Summary
**Health Score:** 🔴 35/100 (Critical Action Required)
The application is currently failing major compilation and linting checks. There are significant type mismatches across the Supabase data access layer and numerous potential runtime errors.

### 1. Build & Type Safety (Critical)
The project has **3,584 TypeScript compilation errors** preventing a successful strict build (`npm run typecheck`).
- **Supabase Query Builder Types:** Files like `src/api.ts` have extensive typing mismatches with the `@supabase/supabase-js` library. Operations like `.eq()`, `.gte()`, `.lte()`, and returning `.data`/`.error` are failing type checks due to incorrect `PostgrestQueryBuilder` typing configurations.
- **Auth Typings:** `src/supabase.ts` is failing because internal data transformations do not conform to `AuthResponse`, `UserResponse`, or `OAuthResponse` standard types.
- **Null Safety:** Multiple UI components (e.g., `src/proforma-invoices/pages/ProformaListPage.tsx`) have `Type 'null' is not assignable to type 'string | undefined'` errors, which can lead to runtime crashes if a value is unexpectedly null from the database.
- **Missing Properties/Imports:** Several files (like `src/templates/StandardTemplate.ts` and `src/utils/pdfGenerator.ts`) are calling methods (`getNumberOfPages()`) that do not exist on the configured types.

### 2. Code Quality & Linting (Major)
The codebase has **1,457 ESLint errors**. 
- These range from unused variables, missing dependency arrays in `useEffect` hooks, to improper React component capitalization.
- Excessive use of `any` types (e.g., `src/reports/invoiceApi.ts` and `src/utils/pdfGenerator.ts`) undermines TypeScript's safety guarantees and hides potential bugs.

### 3. Security Vulnerabilities (Major)
*Note: Transferred from previous security audit.*
- **[PATCHED] Missing Role-Based Access Control (RBAC):** `src/App.tsx` routes users to sensitive admin pages (e.g., `/settings/organisation`, `/settings/access-control`) without verifying if the user has an `admin` role. *Update: This has been patched by implementing an `<AdminRoute>` wrapper.*
- **[PATCHED] Stored XSS via `dangerouslySetInnerHTML`:** Found in `TemplateSettings.tsx`, `DCList.tsx`, `QuotationView.tsx`, and `VerticalTemplate.tsx`. User-generated content is being injected directly into the DOM without sanitization (e.g. DOMPurify). *Update: This has been patched using DOMPurify.sanitize().*

### 4. Code Smells & Anti-patterns (Minor)
- **Extensive `console.error` usage:** There are over 100 scattered `console.error` calls used for error handling instead of a unified logging/toast system in production.
- **Local Storage Misuse:** `localStorage` is used to persist application state logic like `dc_allow_insufficient_stock`. This data is vulnerable to client-side manipulation.

## Reproduction Steps for Build Failures
1. Run `npm run typecheck` to see the 3,584 compilation errors.
2. Run `npm run lint -- src/` to see the 1,457 ESLint errors.

## Next Steps

2. **Security Hardening:** Add a role-check wrapper around admin routes in `App.tsx`.
