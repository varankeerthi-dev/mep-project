Purchase Module: End-to-End Flow (Implemented)

1) Indent / PR Create
From Project / Central / Site Work.
Multi-line items, qty, est. rate, assignment type.
→
2) Approval + Audit
Release workflow (submit/approve/reject).
Full audit trail per transition.
→
3) Material Check
Source determination per line:
  • FULFILL_FROM_STORE
  • PROCURE
Open qty tracked.
→
4) Availability Inquiry
For PROCURE lines: multi-vendor availability/date check.
Partial availability + split allocation.
→
5) Purchase Order (PO)
Convert inquiry response to PO.
Links kept: PR line ↔ inquiry line ↔ PO line.
→
6) Goods Receipt (GR)
Post GR (partial/full).
Updates received qty + PR/PO open qty.
→
7) Bill Entry
Purchase Bill with item lines, tax, totals, vendor link.
→
8) Invoice Verification (3-Way)
Bill vs PO vs GR check with tolerances:
  • Qty %
  • Value %
  • Date days
Status: PASSED / WARN / FAILED.
→
9) Payment Queue + Payment
Pay full/partial, update bill status and vendor ledger.
→
10) Close / Monitor
Dashboards show pending PR, open qty, inquiry splits, IV mismatches, unpaid bills.
Core chain: PR → Inquiry → PO → GR → Bill → IV → Payment
Store path: PR → Store Issue/Receive (no PO needed for available stock).
